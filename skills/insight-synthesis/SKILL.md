# Insight Synthesis

## Goal

Transform raw inputs into a small set of actionable opportunities. Avoid producing a feature inventory.

This bridges Step 2 competitor research into Step 3 creative divergence.

## Step-by-Step Gate

Before jumping to solution directions, present 2-3 possible insight framings with pros, cons, and a recommendation. Examples:

- User journey framing: best when the issue spans before/during/after meeting.
- Role/permission framing: best when host, participant, guest, or admin behavior differs.
- Trust/AI-risk framing: best when summaries, automation, recording, privacy, or source traceability matter.

Ask the user which framing to deepen unless the answer is obvious from the input.

## Inputs

Use whatever is available:

- Need frame.
- Step 1 Q1/Q2/Q3.
- Competitor research.
- Screenshot analysis.
- User feedback or data.
- Existing Tencent Meeting flow or constraints.
- Product/design discussion notes.

## Synthesis Moves

- Cluster facts by user problem, not by competitor.
- Map insights back to Step 1: what supports Q1, what strengthens Q2, and what changes Q3.
- Identify tension pairs, such as speed vs control, automation vs trust, visibility vs privacy, simplicity vs power.
- Separate table-stakes patterns from differentiation opportunities.
- Identify where Tencent Meeting should copy a proven pattern, adapt it, or intentionally diverge.
- Convert observations into "design opportunity" statements.
- Mark opportunities as P0/P1 when the evidence supports prioritization.

## Output

Provide:

- Evidence summary.
- 3-5 key insights.
- Opportunity areas.
- Design principles for this need.
- Which opportunities should enter creative divergence.
- Risks and unknowns.
- Recommended next skill: `solution-exploration`.
