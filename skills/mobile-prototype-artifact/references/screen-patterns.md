# Tencent Meeting Mobile Screen Patterns

## Pre-Meeting Flow

Use when the problem happens before joining.

Recommended screens:

- Meeting detail.
- Join lobby.
- Permission priming.
- Waiting room or admission.

Key states:

- Camera off.
- Mic muted.
- Permission missing.
- External meeting warning.
- Enterprise policy restriction.

## In-Meeting Flow

Use when the problem happens during a live meeting.

Recommended screens:

- Main meeting canvas.
- Expanded controls or more actions.
- Bottom sheet for the selected feature.
- Important feedback state.

Key states:

- Active speaker.
- Screen sharing.
- Recording on.
- AI summary on.
- Weak network.
- Host-only action disabled for participant.

## AI Summary Flow

Use when the work involves AI notes, transcript, summary, action items, or questions.

Recommended screens:

- In-meeting AI entry point.
- Compact AI summary preview.
- Full AI summary sheet.
- Post-meeting summary detail.

Key states:

- Generating.
- Ready.
- Low confidence or incomplete.
- Hidden from guests.
- Disabled by admin policy.

## Collaboration Flow

Use when the work involves chat, reactions, files, comments, or follow-up.

Recommended screens:

- In-meeting chat or comments.
- Mention or action item creation.
- Follow-up summary.
- Share destination.

Key states:

- No messages.
- New unread message.
- Permission blocked.
- Send failure.

## Admin Or Trust Flow

Use when the work involves host controls, privacy, external guests, recording, or enterprise policy.

Recommended screens:

- Trust status overview.
- Permission or policy bottom sheet.
- Host action confirmation.
- Participant-facing notification.

Key states:

- External guest joined.
- Recording started.
- AI enabled.
- Admin policy blocked.
- Host transferred.
