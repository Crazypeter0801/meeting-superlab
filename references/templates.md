## Clarification Questions

```text
我先不直接出方案，先把需求问清楚；这些答案会决定后面竞品、方案和是否出图。请尽量回答下面几个关键问题：

1. 主要用户是谁：主持人、参会者、外部嘉宾、企业管理员，还是其他角色？
2. 问题发生在哪个阶段：会前、会中、会后、管理端、跨设备，还是异步协作？
3. 当前用户路径或产品行为是什么？
4. 现在最核心的痛点是什么：效率、理解、信任、控制、协作、转化，还是其他？
5. 希望这次探索改善什么结果？
6. 如果不做，当前会继续产生什么代价或风险？
7. 怎么判断问题被解决：正向指标、负向指标、止损线、定性体验标准分别是什么？
8. 有哪些约束需要先考虑：权限、隐私、AI 准确性、跨端一致性、企业规则、排期、现有组件？
9. 你手上有什么材料：截图、PRD、会议纪要、数据、用户反馈、竞品链接？
10. 这一轮你希望产出什么：需求澄清、竞品分析、方案方向、demo prompt，PRD check，还是评审材料？
```

## Problem Definition Challenge

```text
下面是我对一个腾讯会议产品需求的 Step 1 问题定义：

Q1 解决谁的什么问题：
Q2 不做会怎样：
Q3 如何衡量问题被解决：

请你作为产品负责人挑战这三个回答：
1. Q1 是否足够具体，是否包含具体用户 / 场景 / 卡点？
2. Q2 是否在证明“不做的代价”，而不是用“做了的好处”反推？
3. Q3 是否同时包含正向指标、负向指标、止损线和定性体验标准？
4. Q1-Q3 是否一致，指标是否真的能验证问题被解决？
5. Kill Criteria 是否能写成 [时间窗口] + [具体指标] + [触发动作]？

请逐条提出质疑，并给出我需要采纳 / 不采纳 / 待确认的建议。
```

## Single Competitor Card

```text
我正在分析竞品 [竞品名称] 在 [具体功能] 上的设计。
我的 Step 1 核心问题是：[贴入 Q1-Q3]

请按 5 个维度输出竞品卡片：
1. 基础信息：竞品名、对应功能、1-2 句话定位
2. 核心机制：用户从入口到结束的完整链路
3. 关键设计点：拆 3-5 个设计决策，每个回答：是什么（位置/文案/状态/行为）/ 为什么 / 价值或风险
4. 数据/反馈：用户量、参与率、评价、吐槽、处罚、负面案例等；信息不足请标注需要继续验证
5. 启示+建议：每个设计点标注“可借鉴 / 需本地化 / 反面教材”，并对应到我的 Step 1 问题

要求：抽象描述不算分析，必须有具体细节；不要编造事实。
```

## Trade-off Matrix

```text
请基于以下 2-3 个候选解法做 Step 4 方案收敛。

Step 1 Q3：

候选解法：
A：
B：
C：

请输出：
1. 五维度 Trade-off Matrix：简洁性 / 成本 / 收益 / 可扩展性 / 风险
2. 每格必须有具体说明，不只给分数
3. 决策路径：
   - 哪些方案有硬伤
   - 回到 Q3，本轮决定性维度是什么
   - 如果选择轻量 v1，需要保留哪些未来扩展约束
4. 最终推荐 v1 方向
5. 为什么选它而不是其他方案
6. 未选方案有哪些优点要保留
```

## PRD AI Check

```text
下面是一份 PRD v0.5：
[贴入 PRD]

Step 1 Q1-Q3：
[贴入]

Step 4 决策记录：
[贴入]

请完成两件事：

第一部分：按 11 项清单逐项检查，每项给出 Green / Yellow / Red / N/A + 具体发现：
1. 主路径优雅简洁性
2. 分支完整性
3. 新手引导
4. 端兼容
5. 多语言
6. 大字号
7. 深色模式
8. 弱网表现
9. 灰度逻辑 + A/B 实验
10. 监控指标
11. Kill Criteria 完整性

对 Yellow 或 Red 的发现，给出具体问题、为什么重要、可能修订方向、影响对象。

第二部分：扫描所有用户可见文案，输出中 / 英 / 繁多语言对照表。

最后输出可放入 PRD 顶部修订记录的表格：
日期 / 修订来源 / Check 项 / 评级 / 修订内容 / 修订人
```

## Low-Fidelity Demo Prompt

```text
请基于以下方案生成会议产品体验的黑白低保真 demo。

目标：

用户角色：

使用场景：

核心流程：

页面/状态：
1. 正常状态
2. 空状态
3. 权限或异常状态
4. 弱网或超时状态
5. 多角色或跨端状态

设计要求：
- 黑白低保真
- 不要过度视觉风格
- 聚焦信息结构、操作路径、关键状态
- 使用真实但简洁的会议产品文案
- 可以借鉴竞品的交互思路，但不要复制竞品视觉

请输出页面结构、关键控件、用户路径和评审时需要关注的问题。
```

## Figma Handoff Question

```text
这个方案已经可以进入低保真稿阶段。要不要我帮你画到 Figma 里？

如果需要，请发我：
1. Figma 文件或具体 node 链接
2. 希望新增页面、放在某个节点旁边，还是更新已有稿
3. 画稿范围：Tiny flow / Review-ready flow / Concept flow
4. 保持黑白低保真，还是做更接近腾讯会议产品稿的样式
5. 是否需要复用文件里的现有组件或设计系统
```
